XTP Pro版本的资金划拨是在资金划拨服务里完成操作的，用户发出资金划拨请求之前，需确保当前资金划拨服务是可用状态。 下面实例代码从Login开始，到收到资金划拨服务的通知OnServerStatusNotification()，再做资金划拨，以及资金划拨各种情况下的参数设置示例。   


## **1\. 总体步骤说明** ​

### 1.1. 继承Spi ​

代码通过#include "xtpx_trader_api.h"，将xtpxtraderapi.lib中声明的类和数据类型包括进来，该头文件中有两个重要的基类：**TraderSpi** 和**TraderApi** 。

TraderApi类提供交易相关的请求接口，本示例中调用登录接口和资金划拨接口。

TraderSpi类提供交易相关的回调接口，本示例需要继承该类并重写OnServerStatusNotification()回调接口和OnFundTransfer()回调接口。

### 1.2. 初始化Api ​

在主函数main()里完成MyTraderApi的初始化，创建TraderSpi实例，并向TraderApi实例注册；

### 1.3. 登录交易服务器 ​

登录交易服务器，登录前设置必要参数，包括设置软件开发版本号，软件开发Key，心跳检测时间间隔等，登录接口返回会话ID表示登录成功；

### 1.4. 监测资金划拨服务是否可用 ​

登录交易服务器后，api端内部会自动连接资金划拨服务，如果连接成功，会收到服务状态变化的通知。示例如下：

cpp
    
    
    void MyTraderSpi::OnServerStatusNotification(uint64_t session_id, uint32_t server_type, bool status)
    {
    	if (1 == server_type && status)//server_type:1表示资金划拨服务，2表示查询服务
    	{
    		requestFundTransfer(pUserApi, session_id);//资金划拨服务可用时，发起资金划拨请求
    		
    	}
    	
    }

### 1.5. 调用资金划拨接口FundTransfer ​

登录交易服务器后，并收到OnServerStatusNotification()的资金划拨服务可用状态后，就可以用api调用资金划拨请求接口。

注意：实盘中资金划拨请求有频率限制，1秒内请求不超过1次，超出会报错。

各种资金划拨参数设置示例代码：

### 1.6. 重写资金划转的回调函数逻辑 ​

资金划拨成功后，划拨结果通过回调函数OnFundTransfer()返回，用户根据自己的策略重写该回调函数。

cpp
    
    
    void MyTraderSpi::OnFundTransfer(XTPFundTransferNotice *fund_transfer_info, XTPRI *error_info, uint64_t session_id)
    {
        string message;
        if (error_info != NULL && error_info->error_id != 0)
    	{
    		//划转失败
    		message = string(" fund transfer is failed:") + to_string(error_info->error_id) + ": " + error_info->error_msg + "\n";
    	}
    	else
    	{
            //TODO:可以根据GetClientIDByXTPID() == client_id来过滤自己的订单
            uint8_t temp_client_id = pUserApi->GetClientIDByXTPID(fund_transfer_info->serial_id);
            if (temp_client_id == client_id)//注意client_id为本程序中创建TraderApi时传入的client_id，用户需要自己定义修改
            {
                //TODO：此资金划拨订单由本程序发起的
            }
            else
            {
                //TODO：此资金划拨订单非本程序发起
            }
    
            //根据资金划拨订单的状态来处理自己的资金逻辑
            switch(fund_transfer_info->oper_status)
            {
                case XTP_FUND_OPER_SUCCESS:
                {
                    //TODO:根据资金划拨的方向，去做后续处理
                    if(fund_transfer_info->transfer_type == XTP_FUND_INTER_TRANSFER_IN)
                       message = string(" internal fund transfer is success: fund amount:") + to_string(fund_transfer_info->amount) + "\n";
                    break;
                }
                case XTP_FUND_OPER_FAILED:
                {
                    message = string(" fund transfer is failed.\n");
                    break;
                }                
                case XTP_FUND_OPER_PROCESSING:
                {
                    message = string(" fund transfer is processing.\n");
                    break;
                }
                case XTP_FUND_OPER_SUBMITTED:
                {
                    message = string(" fund transfer is arrived at the counter to centralized process.\n");
                    break;
                }
                default:
                    break;
    
            }
            	
    	}
        cout << message;
    }
